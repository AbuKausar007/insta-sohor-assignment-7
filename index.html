(১). জাভাস্ক্রিপ্ট কিভাবে কাজ করে?

উত্তর: প্রতিটি ব্রাউজারে জাভাস্ক্রিপ্ট কোড চালানোর জন্য একটি ইঞ্জিন আছে। জাভাস্ক্রিপ্ট ইঞ্জিন হল একটি কম্পিউটার প্রোগ্রাম যা জাভাস্ক্রিপ্ট কোডকে রান করে এবং এটিকে কম্পিউটারের বোধগম্য ভাষায় রূপান্তর করে। উদাহরণস্বরূপ, গুগল ক্রোমে রয়েছে "V8 ইঞ্জিন", মজিলা ফায়ারফক্সে রয়েছে "স্পাইডার মাংকি"। যখনই আমরা একটি জাভাস্ক্রিপ্ট প্রোগ্রাম চালাই, তখন ইঞ্জিনে একটি "পার্সার" থাকে যা আমাদের কোডটিকে রিসিভ করে। পার্সার জাভাস্ক্রিপ্ট কোডটিকে লাইন বাই লাইন চেক করে খুঁজে বের করতে চেষ্টা এতে কোন সিনট্যাকটিক ত্রুটি আছে কি না। যদি কোডে কোনো ত্রুটি না থাকে, তাহলে এটি AST (Abstract Syntax Tree) নামে একটি ডেটা স্ট্রাকচার তৈরি করে। যখন পার্সার দ্বারা AST তৈরি করা হয়, তখন JavaScript ইঞ্জিন জাভাস্ক্রিপ্ট কোডটিকে মেশিন ভাষায় (বাইট কোড) রূপান্তর করে যা কম্পিউটার বুঝতে পারে। তারপর এই রূপান্তরিত কোডটি কার্যকর করার জন্য সিস্টেমে পাঠানো হয়।


(২) Local storage & Session storage এর মধ্যে পার্থক্য কি?

উত্তর: Local storage এবং Session storage ওয়েব স্টোরেজ API নামে পরিচিত। এই API গুলি ব্যবহার করে ক্লায়েন্ট সাইডে ডেটা সংরক্ষণ করা যেতে পারে। Local storage ব্রাউজারে ব্যবহারকারীর ডেটা স্থায়ীভাবে সংরক্ষণ করে। এটির কোন মেয়াদ শেষ হওয়ার সময় নেই, Local storage-এ ডেটা বিদ্যমান থাকে যতক্ষণ না ব্যবহারকারী ম্যানুয়ালি এটি মুছে ফেলেন। ব্রাউজার রিস্টার্ট হওয়ার পরেও Local storage-এ ডেটা থেকে যায়। ব্রাউজারের যেকোনো উইন্ডোতে ডেটা সেট করলে Local storage এর ক্ষেত্রে তা সমস্ত উইন্ডোর মধ্যে শেয়ার হয়ে যায়, তাই আমরা যদি একটি উইন্ডোতে ডেটা সেট করি, তখন অন্য একটি উইন্ডোতেও সেটা দৃশ্যমান হয়। Session storage ব্রাউজারে একটি নির্দিষ্ট সময়ের জন্য ব্যবহারকারীর ডেটা সংরক্ষণ করে। Curretn window বন্ধ না হওয়া পর্যন্ত Session storage-এ ডেটা বিদ্যমান থাকে। যদি আমরা Curretn window বন্ধ করি তাহলে আমাদের ডেটা Session storage থেকে স্বয়ংক্রিয়ভাবে মুছে যাবে। Session storage, window থেকে window-তে পরিবর্তিত হয়। একই ওয়েবসাইটের আরেকটি ুউইন্ডতে আলাদা স্টোরেজ থাকে Session storage এর ক্ষেত্রে। কিন্তু উভয় স্টোরেজ একই Methods ব্যবহার করে থাকে। যেমনঃ setItem(), getItem(), removeItem() ইত্যাদি।
